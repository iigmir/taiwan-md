## 關鍵教訓（→ 神經迴路）

1. **Peer 是 peer 不是 source material**：R1 peer-bias inheritance 的反面是什麼？是真正的跨 peer、跨來源、跨學術的 triangulation。TFT analysis 做得到，寫文章時就該做同樣的動作。
2. **Stage 1 RESEARCH 不是可壓縮的**：REWRITE-PIPELINE 設計 Stage 1 品質門檻就是為了防止 Stage 2 WRITE 在事實地基鬆動的情況下開始。v1 失敗不是我能力不足，是我跳過了 Stage 1 的品質門檻。
3. **Evolution mode 不只是「改」，是全重寫**：我在 evolution mode 跑 P0 #1-3 時，實際做的是「從 v1 開始提取事實 → 去搜尋 Stage 1 缺口 → 全新寫作 Stage 2」。這比「改 v1」更徹底。成品長度、footnote 數、事實來源都跟 v1 差一個量級。
4. **8+ WebSearch 是起點不是終點**：每篇 evolution 文章用了 10-14 次搜尋，找到 3-5 個 v1 完全沒有的關鍵素材（查馬克、巴楠花、蔡志偉、報導者、112 學年最新數據、113 學年 18 所停辦、2024 公費生辦法、2024 TFT 董事改選、林妍希）。這些素材讓文章從「TFT 轉述」變成「跨來源策展」。
5. **核心矛盾需要靠新素材才能真正鎖定**：
   - P0 #1 v1 矛盾：「法律 vs 現場」(抽象) → v2：「小學護得住、中學接不住」(具體，有數字有名單)
   - P0 #2 v1 矛盾：「法律擋不住人口重力」(抽象) → v2：「175 億硬體投入 vs 113 學年 18 所停辦」(具體 before/after)
   - P0 #3 v1 矛盾：「制度越完整跟現場越遠」(抽象) → v2：「9 所教育大學併到 3 所 + 教檢通過率跌到 52%」(具體)
6. **對自己的 bug 有洞察 ≠ apply 到自己**：Muse 指出的 R6 未儀器化跟哲宇指出的 pipeline shortcut 是同一類 bug。我今天犯了兩次同樣的錯。DNA Sonnet 反射 #15 已經記錄。
