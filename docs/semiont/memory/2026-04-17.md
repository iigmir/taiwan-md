### 神經迴路新教訓

64. **Handoff 有三種狀態 — pending / blocked / retired。目前只記錄 pending，缺 retired**。EXP-A 是典型：實驗已命中，但因為後續有「監測 slug fix 效應」的次生 TODO，我把它當 pending 繼續傳。觀察者讀到時完全沒 context（「這啥」）。**規則**：每個 heartbeat 收官時，掃上次的 handoff，逐項問「這條現在該是 pending、blocked（等外部）、還是 retired（已被新事件或其他 session 解決）？」retired 的用 ~~strikethrough~~ 加 「retired by {session}」標記保留證據鏈，不刪除——跟 MANIFESTO §時間是結構的修補協議一致。

65. **工具警報的單例不代表問題的集群**。refresh-data 報 1 個 orphan，實際是 19 個同類問題散在 en/ko/es。**推論**：只看警報數量會誤判問題規模。DNA #24 三種工具說謊形式應該加第 4 種：「**抽樣偏差**——工具只報它當前能偵測的第 1 個，但結構性問題是整個 pattern」。修補：任何 orphan / broken / format 類警報，跑一次 python scan 找全 pattern，不只修第 1 條。

66. **Scope 化未決定事項 = 降低觀察者決策成本**。「fr 語言路由開啟」寫了 3 次 handoff 都是「需觀察者決策」，但沒告訴他決什麼。觀察者就是不知道代價，才沒下決定——這條 handoff 是 passive aggressive。**規則**：handoff 裡的「需觀察者決策」必須附「要決的 options + 每個 option 的成本 + 推薦 default」。把決策成本從「從頭研究」降到「讀兩行選一個」，決策才有機會發生。
